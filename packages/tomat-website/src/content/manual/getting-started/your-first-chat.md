A **session** is one conversation with the agent. You type in the bar at the
bottom; the agent replies in a **bubble** above it, exactly like the demo here.

## Sending a message

Type your message and press Enter. The agent streams its reply as it thinks. If
you have set up a local model, the whole exchange happens on your machine.

## Reading replies

- **Your messages** sit on one side, the **agent's** on the other.
- When the agent thinks before answering, a **Thinking** bubble shows how long
  it spent before the reply lands.
- Tool runs and other steps appear inline so you can follow what the agent did.

## Starting a new session

Open the session list and start a new one whenever you want a clean context.
Old sessions stay saved locally so you can return to them.
