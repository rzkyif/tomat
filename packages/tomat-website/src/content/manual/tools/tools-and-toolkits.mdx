A **tool** is one thing the agent can do beyond talking: search the web, fetch a
page, run a calculation. A **toolkit** is a bundle of related tools that you
install or remove as a unit. Tools are how the agent acts in the world rather than
only describing it. A fresh core comes with a built-in toolkit already installed,
so the abilities are there to switch on, with room to add more.

## How tools reach the agent

You rarely hand-pick tools for a message. Before each one, tomat ranks your
installed tools by how relevant they are to what you just asked, and offers the
agent only the best matches. A model handed every tool you own would waste its
attention sorting through them; a short, relevant list keeps it focused on the
task.

<RelevantToolsDemo />

A few tools you rely on regardless can be marked as always available, so they are
offered every time instead of waiting to be ranked in. The result, the ranked
shortlist plus your always-on tools, is what the conversation shows you for each
turn, as covered in [Inline Results](/manual/conversations/inline-results).

## Managing toolkits

Tool use is off until you turn it on under **Tools** in Settings, so a fresh
install talks but does not act until you let it. The same group is where you
install, remove, and configure toolkits, and holds the dials for the selection
above: how many tools tomat offers per message, and which tools are always
available regardless of relevance.

<SettingsPanelDemo group="tools" />

Adding a toolkit is how you teach the agent new abilities, and removing one takes
them away cleanly, since a toolkit is a self-contained bundle rather than a loose
pile of tools.

## Permissions and the sandbox

A tool can reach outside tomat, so each toolkit declares the permissions it needs
up front, such as access to a specific network host, and runs in a sandbox that
holds it to them. A tool can only do what you have allowed it to, and you can see
what it asked for before you let it run.

<Aside type="caution">
  A tool that reaches the network or runs commands acts on your behalf. Install toolkits you trust,
  and read the permissions a toolkit declares before granting them.
</Aside>

Tools and your saved knowledge work the same way under the hood: tomat finds what
is relevant and offers it to the agent. The next chapter covers the knowledge
side, starting with [Memories](/manual/knowledge/memories).
