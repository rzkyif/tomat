When the agent does more than write text, that work shows up inside the
conversation, not behind it. A reply becomes a record of what the agent actually
did: which tools it considered, what those tools returned, and which of your
memories it drew on. Reading that record is how you trust an answer instead of
taking it on faith.

## Rich tool output

A tool reports its progress as it runs and can render its result right in the
session. Output is not limited to plain text: a tool can show a table, an image,
or a diff, so a change reads clearly instead of as a wall of prose. A diff is the
clearest example, marking exactly what was added and removed.

<DiffDemo />

A tool can also pause to ask you something mid-run, for instance to confirm an
action before it goes ahead, and wait for your answer before continuing. So an
inline result is not always final; sometimes it is the agent checking in.

## Which tools were offered

Before each message, tomat ranks your installed tools by how relevant they are to
what you asked and offers the agent only the best matches. The conversation shows
that selection, so you can see which tools were on the table for a given turn
rather than guessing.

<RelevantToolsDemo />

This keeps the model focused on a short, relevant list, and keeps you informed
about what it could reach for. The ranking, and the tools you mark as always
available, are covered in [Tools and Toolkits](/manual/tools/tools-and-toolkits).

## Which memories were consulted

The same happens with your knowledge. As you write, tomat finds the memories most
related to your message and lets the agent see them, and the conversation shows
which ones were surfaced. You do not have to remember that a relevant memory exists;
tomat brings it forward and tells you it did.

<RelevantMemoriesDemo />

When nothing matches, tomat can stay quiet or show that it looked and found
nothing, whichever you prefer. The retrieval behavior and its settings live in
[Memories](/manual/knowledge/memories).

Together, these inline results turn a reply into something you can audit. To set
up the tools and knowledge behind them, continue to
[Tools and Toolkits](/manual/tools/tools-and-toolkits).
