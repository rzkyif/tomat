The core does tomat's work: it runs models, executes tools, and keeps your
sessions. Most installs run it on the same machine as the client, set up for you
the first time you launch the app. But the core does not have to live where the
client does. Putting it on a more powerful computer lets the heavy model run there
while you sit at a laptop, as [How tomat Works](/manual/core-concepts/how-tomat-works)
describes.

## Install the core

On the machine that will run the core, copy this command into a terminal. The
toggles fold the options you want into the command before you run it.

<CoreInstallCard client:visible />

- **Channel** picks how new a build you run, matching the channel you use on the
  client. Leave it on **Stable** unless you have a reason not to (see
  [Updates and Channels](/manual/settings/updates-and-channels)).
- **Allow access from other machines** is the one that matters for a remote setup.
  By default a core only listens to its own machine, so nothing else on the network
  can reach it. Turn this on to let the client on another computer pair with it.
- **Run in the background at login** installs the core as a service so it starts on
  its own and is always there to connect to. Turn it off if you would rather start
  the core by hand.

<Aside type="caution">
  Allowing access from other machines opens the core to your local network. Only do this on a
  network you trust; pairing still requires the one-time code, but the core is reachable to anything
  that can route to it.
</Aside>

## Pair the client with it

When the install finishes it starts the core and prints a **pairing code**. On the
machine with the client, add a core by its address (the host's name or IP) and
enter that code once; the client stores the trust and reconnects on its own from
then on. The full flow, and how to manage more than one core, is in
[Cores and Pairing](/manual/settings/cores-and-pairing).

A pairing code is short-lived. If it expires before you use it, or you want to pair
another client later, you can mint a fresh one on the core's machine using the admin
token the installer saved under `~/.tomat/<channel>/core`. The command the installer
prints at the end shows exactly how.

Next is the client side of this: how pairing works and how to move between cores in
[Cores and Pairing](/manual/settings/cores-and-pairing).
